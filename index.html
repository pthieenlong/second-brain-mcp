Cài đặt

Không cần clone repo, không cần build. npx tự tải package khi client khởi động.

  1. Kiểm tra Node.js

    Cần phiên bản 20 trở lên. Mở Terminal (macOS) hoặc Command Prompt (Windows) và gõ:

    terminal
    node --version

    Báo lỗi hoặc số nhỏ hơn 20 thì tải tại nodejs.org.

  2. Chọn nơi lưu ghi chú

    Một thư mục bất kỳ, ví dụ /Users/ban/Documents/Notes. Chưa tồn tại cũng được — nó sẽ tự tạo. Đang dùng Obsidian thì trỏ thẳng vào vault sẵn có.

  3. Thêm vào cấu hình MCP của client

    Mở file cấu hình MCP rồi thêm khối này. Chỗ đặt file tùy client — tìm mục “MCP servers” trong tài liệu của họ.

    mcp config · json 
    {
  "mcpServers": {
    "second-brain": {
      "command": "npx",
      "args": ["-y", "@pthieenlong/second-brain-mcp"],
      "env": {
        "VAULT_PATH": "/Users/ban/Documents/Notes"
      }
    }
  }
}

    Đường dẫn phải đầy đủ từ gốc. Dùng ~/Documents/Notes hay ./Notes sẽ lỗi. Nếu file đã có sẵn mcpServers, chỉ thêm khóa "second-brain" vào bên trong, đừng ghi đè cả khối.

  4. Khởi động lại client rồi thử

    Đóng hẳn và mở lại. Nhắn: “Lưu note này vào second brain: hôm nay học được cách dùng React hooks” — thành công thì bạn thấy file .md mới trong thư mục vừa chọn.

    Lần chạy đầu hơi chậm vì npx đang tải package. Những lần sau dùng bản cache nên nhanh.

Ghi chú nằm ở đâu

Mỗi note là một file .md trong VAULT_PATH/<category>/. Đây là bản gốc, là nguồn sự thật — mở bằng Obsidian, tìm bằng grep, đồng bộ, sao lưu đều được. 

Notes/
├─ 00-Inbox/          chưa phân loại
├─ 01-Fundamentals/
├─ 02-Work/
├─ 03-Product-Thinking/
├─ 04-Learning/
├─ 05-Career/
├─ 06-Personal/
├─ 07-Projects/
├─ 08-Canvas/         export_flow
└─ .second-brain/
   └─ index.db        chỉ mục

Đổi danh sách category

Bảy nhóm trên là mặc định. Muốn dùng nhóm riêng, thêm NOTE_CATEGORIES vào env rồi khởi động lại client:

env 
"NOTE_CATEGORIES": "Nau-an,Du-lich,Cong-viec,Y-tuong"

Ngăn cách bằng dấu phẩy, không khoảng trắng thừa. AI chọn nhầm nhóm không có trong danh sách thì note rơi vào 00-Inbox chứ không bị từ chối — bạn sắp xếp lại sau, không mất gì.

File index.db chỉ là dữ liệu phái sinh: một SQLite nhỏ lưu metadata để tìm nhanh, khỏi phải đọc từng file. Xóa đi thì nó tự tạo lại — chạy reindex_vault để nạp đầy.

Chưa làm được gì

Nói thẳng trước khi bạn cài, để không mất thời gian đoán.

Chỉ mục có thể lệch với thư mục
Note được ghi ra file trước, rồi mới đưa vào chỉ mục. Bước sau lỗi thì file vẫn còn nhưng tìm không thấy. Sửa note trực tiếp trong Obsidian cũng không tự cập nhật. Nghi ngờ lệch thì bảo AI “reindex vault giúp tôi”.
Chỉ tìm được tiêu đề, chưa tìm trong nội dung
search_notes khớp theo tiêu đề, category và tag. Chữ nằm trong thân note thì chưa tìm được.
Chữ tiếng Việt có dấu phân biệt hoa thường
Tiêu đề tiếng Anh tìm không phân biệt hoa thường, nhưng tiếng Việt có dấu thì có — hạn chế của SQLite.
Chưa có bản dựng cho Windows ARM64
Driver SQLite (libsql) chưa phát hành binary cho nền tảng này.

Dựng bằng gì

NestJS chạy trên Model Context Protocol SDK, nói JSON-RPC qua stdio. Metadata note lưu trong SQLite qua Prisma 7 với adapter libsql — ship sẵn binary theo nền tảng, nên lúc cài không phải biên dịch gì.

chạy thử không cần client 
$ echo '{"jsonrpc":"2.0","id":1,"method":"tools/list"}' \
  | VAULT_PATH=/tmp/test-vault npx -y @pthieenlong/second-brain-mcp
